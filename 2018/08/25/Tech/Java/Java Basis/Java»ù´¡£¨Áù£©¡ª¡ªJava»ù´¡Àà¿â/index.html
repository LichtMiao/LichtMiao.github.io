Java基础（六）——Java基础类库

Posted on 2018-08-25 | In Tech , Java , Java Basis

Java8提供了4000多个基础类库，通过这些基础类库可以提高开发效率，降低开发难度。对于合格的java程序员而言，至少要熟悉Java SE中70%以上的类，在反复查阅API文档的过程中，会自动记住大部分的功能、方法，因此程序员一定要多练，多敲代码。
Java提供了String、StringBuffer和StringBuilder来处理字符串，它们之间存在少许差别。
使用Calender来处理日期、时间。
正则表达式是一个强大的文本处理工具，通过正则表达式可以对文本内容改进型查找、替换、分割等操作。
Java使用Locale对象封装一个国家、语言环境，在使用ResourceBundle根据Locale加载语言资源包，当ResourceBundle加载了指定Locale对应的语言资源文件后，ResourceBundle对象就可以调用getString()方法来去除指定key所对应的消息字符串。

方法的调用规则：谁调用该方法，谁负责为形参赋值。则main方法由JVM调用，形参args应该由JVM赋值。

Scanner类

Scanner是一个基于正则表达式的文本扫描器，它可以从文件、输入流、字符串中解析出基本类型值和字符串值。它主要提供两个方法来扫描输入：
（1）hasNext*** ()：是否还有下一个输入项，***可以是Int、Long等代表基本数据类型的字符串。如果只是判断是否包含下一个字符串，则直接使用hasNext()。
（2）next***()：获取下一个输入项。

如果不需要Scanner类通过空白键作为分隔符，可以使用useDelimiter(String pattern)方法，该方法的参数是一个正则表达式。如在上面程序的构造器后添加代码：
sc.useDelimiter(“\n”); // 使用回车作为分隔符（de’limiter定界符）
Scanner提供两个简单的方法来逐行读取：
(1) boolean hasNextLine ()：返回输入源中是否还有下一行。
(2) String nextLine ()：返回输入源中下一行的字符串。

系统相关（System类、Runtime类）

Java提供了System类和Runtime类来与程序的运行平台进行交互。
System类代表当前Java程序的运行平台，程序不能创建System类的对象。
System类提供了代表标准输入、标准输出和错误输出的类变量，并提供了一些类方法用于访问环境变量、系统属性的方法，还提供了加载文件和动态链接库的方法。

System类

以下程序通过调用System类的getenv()、getProperties()、getProperty()（这个方法是getProperties()方法的特例形式，专门获取某些方法）等方法来访问程序所在平台的环境变量和系统属性，程序运行结果会输出操作系统所有的环境变量值，放在项目根目录文件中F:\eclipsework\Crazy-java。

System类的in、out和err分别代表系统的标准输入（通常是键盘输入）、标准输入（通常是显示器输出）和错误输出流。并提供了setIn()、setOut()和setErr()方法来改变系统的标准输入、标准输出和标准错误输出流。
System类提供identityHashCode(Object x)方法来访问指定对象的精确HashCode值（可以用来判断是否是同一对象，==也有此功能）。当某个类的HashCode()方法被重写，HashCode()方法就不能唯一地标识该对象；但通过identityHashCode()方法返回的HashCode值是根据该对象的地址得到的HashCode值。如果调用该方法得到的identityHashCode值是一样的，则两个对象一定是同一个对象。

注意：栈内存中存放基本类型变量和引用类型变量；Java堆用来存放new创建的对象和数组；方法区用来存放常量和静态变量。具体请见Java虚拟机（一）——Java内存区域与内存溢出异常

Object类

Object类是所有类、数组、枚举类的父类。所有java类都是Object类的子类，所以任何java对象都可以调用Object类的方法。
Protected void finalize()
Class<?> getClass()：返回该对象的运行时类。
int hashCode()：返回该对象的hashCode值。
String toString()：返回该对象的字符串表示。Object类的toString()方法返回的是“运行时类名@十六进制hashCode值”的字符串。但很多类都重写了toString()方法，用户返回表述该对象信息的字符串。
wait()、notify()、notifyAll()：控制线程的暂停与运行。
protected修饰的clone()方法：帮助其他对象来实现“自我克隆”，得到当前对象的副本，而且二者之间完全隔离。protected修饰：只能被子类重写或调用。自定义类实现“自我克隆”的步骤：

1) 自定义类实现Cloneable接口
2) 自定义类实现自己的clone()方法。
3) 通过super.clone()实现clone()方法。（调用父类的clone()方法实现自我克隆）

注意：Object类的clone()方法虽简单、易用，但它只是克隆对象的所有成员变量的值，不会对引用类型的成员变量值所引用的对象进行克隆。
Java7新增的Objects工具类，它提供了一些工具方法。Java为工具类的命名习惯是添加字母s，如操作数组的工具类Arrays，操作集合的工具类Collections。

上述程序还示范了Objects提供的requireNonNull()方法，当传入的参数不为null时，返回参数本身，否则将引发NullPointerException异常。

String类

1) String类、StringBuffer类、StringBuilder类。
String类是不可变类。
2) StringBuffer（线程安全）对象则代表一个字符序列可变（内容可变，长度可变）的字符串。当一个StringBuffer对象被创建以后，可以通过StringBuffer提供的append()、insert()、reverse()（追加、插入、相反（倒转））、setCharAt()、setLength()等可以改变这个字符串对象的字符序列。一旦通过StringBuffer生成了最终想要的字符串，就可以调用它的toString()方法将其转换为一个String对象。
3) StringBuilder类（线程不安全，适合单线程）也是代表字符串对象，与StringBuffer类的使用类似。不同是StringBuffer类是线程安全的，而StringBuilder类没有实现线程安全功能，因此性能略高。（线程安全是指某个函数、函数库在多线程环境中被调用时，能够正确地处理多个线程之间的共享变量不致发生混乱，使程序功能正确完成）。因此在通常情况下，如果需要创建一个内容可变的字符串对象，则应该优先考虑使用StringBuiler类。

String、StringBuilder、StringBuffer类都实现了CharSequence接口，因此CharSquence接口可认为是一个字符串的协议接口。

String(byte[] bytes, int offset, int length, String charsetName)
int compareTo(String anotherString)：比较两个字符串的大小。如果相当则返回0；不相等时，从两个字符串第0个字符开始比较，返回第一个不相同的字符值之差（this.charAt(i) – anotherString.charAt(i)）。另一种情况是，长字符串前面部分刚好是短字符串，则返回它们的字符个数差（this.length() – anotherString.length()）。
int length()：返回当前字符串长度。
byte[] getBytes()：将该String对象转换成byte数组。
char[] toCharArray()：将该String对象转换成char数组。
StringBuilder、StringBuffer类与String类不同的是，它提供length()来访问字符序列长度、提供setLength()方法来修改器字符序列长度。

Java7的ThreadLocalRandom和Random

ThreadLocalRandom类与Random类的用法类似，它提供一个current()方法获取当前ThreadLocalRandom对象（随机对象），获取该对象之后即可调用各种next ()方法来获取伪随机数了。
byte[] buffer = new byte[16];
buffer.nextBytes();
Random使用一个48位的种子，当两个Random对象种子相同时，它们会产生相同的数字序列。即Random产生的数字并不是真正随机的，而是一种伪随机。为了避免两个Random对象产生相同的数字序列，通常推荐使用当前时间作为Random对象，如：
Random rand = new Random(System.currentTimeMillis()); （millisecond毫秒）
在多线程环境下使用ThreadLocalRandom的方式与使用Random基本类似，如：
ThreadLocalRandom rand = new ThreadLocalRandom.current();
//生成一个420之间的伪随机数
int val1 = rand.nextInt(4,20);
//生成一个2.010.0之间的伪随机数
int val2 = rand.nextDouble(2.0,10.0);

BigDecimal类

import java.math.BigDecimal;
float、double这两种基本类型浮点数容易引起精度丢失。
BigDecimal类，通常优先使用基于String的构造器。如new BigDecimal(“0.1”);
如果必须使用double浮点数作为BigDecimal构造器的参数时，不要直接将该double浮点数作为构造器参数创建BigDecimal对象，而应该通过BigDecimal.valueOf(double value)静态方法来创建BigDecimal对象。
BigDecimal类对精确浮点数进行常规运算的方法：add()、subtract()、multiply()、divide()、pow()等。
例：
BigDecimal f1 = new BigDecimal(“0.05”);
BigDecimal f2 = BigDecimal.valueOf(0.05);

Calendar类

Import java.util.Calendar;
Calendar类是一个抽象类，所以不能使用构造器来创建Calendar对象。可以使用它提供的静态方法getInstance()来获取Calendar对象。
Calendar calendar = Calendar.gerInstance();
注意：第一个月从0开始，因此需要get(Calender.MONTH) + 1。
add与roll的区别：add修改的字段超过它允许的范围时，会发生进位。如果下一字段也需要改变，则该字段会修正到变化最小的值。
而roll上一级字段不会进位，下一级字段修正规则与add类似。
（1）默认是设置了容错性的，setLenient()方法用于设置Calendar类的容错性，防止产生不合理的输入值导致进位或减位。
（2）要关闭容错性使用setLenient(false); 此时让程序进行严格的参数检查。程序会抛出异常。
set()方法延迟修改，尽管日历字段通过set()方法后会被立即更改，但Calendar所代表的时间却不会立即更改，它只是使用内部变量表记录这个被修改字段的值，直到调用get()、getTime()、getTimeInMillis()、add()、roll()方法时，才会重新计算日历时间。这样，多次调用set()方法不会产生不必要的计算。
Java8新增了java.time包。

正则表达式（regular expression）

//regex正则表达式
import java.util.regex.Pattern;
import java.util.regex.Matcher;

是一个强大的字符串处理工具，可以对字符串进行查找、提取、分割、替换等操作。
Java专门提供了Pattern和Matcher类用于提供正则表达式支持。
对于字符：\\w{3,20}@\\w+\.(com|org|cn|net|gov)， 花括号表示单词长度320，括号表示选择。
Java字符串中反斜杠本身需要转义，如若要输出”?”，需要写”?”，但由于反斜杠本身需要转义，则最终写成”\?”。前一个反斜杠用于转义，后一个反斜杠用于匹配。
Java正则表达式中还可以使用“通配符”：预定义字符。一般小写表示这种类型字符，大写表示非这种类型的字符。\w：单词字符，包含字母或数字。
方括号表达式：若只想匹配af的字母，或者匹配中文字符。方括号一般用来限定一个范围。
